Introduction

The Soil Health Analyzer is a web application that analyses soil images and returns a soil health score and diagnostics (texture, moisture indicator, and basic nutrient inference). It consists of a React front-end and a Node/Express backend that hosts the analysis model or forwards images to an ML service.

Quick Start

Prerequisites

  • Node.js v14+ and npm
  • Modern browser (Chrome, Firefox, Edge)
  • Optional: Python environment if ML runs in a Python service

Install & Run

        # from project root
        npm install
        npm run dev   # starts the React frontend

        # in /server
        cd server
        npm install
        node index.js    # or npm run start

Notes

  • By default the front-end expects the backend API at /api. Adjust src/config or environment variables if needed.
  • Replace placeholder ML endpoints with your deployed model URL or local Python microservice.

How Soil Analysis Works

  1. User uploads an image via the web UI.
  2. Front-end sends multipart/form-data to /api/analyze.
  3. Backend stores the image temporarily and forwards it to the ML model/service (local or remote).
  4. ML model inspects color channels, texture patterns, and optionally runs a trained classifier/regressor to predict moisture/nutrient hints.
  5. Backend composes a JSON result and returns it to the front-end.

Deployment

Suggested deployment pattern:

  1. Build front-end (npm run build) and serve from a static host (Netlify, Vercel) or behind the backend as static files.
  2. Host backend on Heroku / Render / VPS. If ML model is heavy, run it in a separate Python service or container and expose a gRPC/HTTP endpoint.
  3. Use cloud storage for images and secure the API with API keys / JWT authentication.

Quick Tips

  • Validate images on the client (size/type) before upload.
  • Use async worker queues for long-running ML tasks (Bull, Redis).
  • Cache recent results to reduce repeated analyses.
